# Cro

## Workflow
1. Get the current page/form (URL, screenshot, or description) and the specific conversion goal + current baseline if known.
2. Audit against CRO fundamentals:
   - Value proposition clarity above the fold
   - Friction in the form (number of fields, unnecessary required fields)
   - Trust signals (social proof, security badges, testimonials) placement
   - CTA clarity and visual prominence (one primary action per screen)
   - Mobile experience specifically
3. Propose 3-5 concrete, testable changes ranked by expected impact vs effort, framed as hypotheses ("if we do X, we expect Y because Z") suitable for A/B testing.
4. Avoid dark patterns (fake urgency, hidden costs, forced continuity) even if asked - note the ethical/legal risk instead.
