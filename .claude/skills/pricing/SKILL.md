# Pricing

## Workflow
1. Get the product, target customer segments, and cost structure/margin needs if known.
2. Choose a pricing model fit to the value metric (per-seat, usage-based, flat-tier) - the metric should scale with the value the customer gets, not arbitrarily.
3. Design 2-4 tiers: what moves someone up a tier should be a real usage/value threshold, not an arbitrary feature lock that annoys users.
4. Sanity-check against comparable products' public pricing (search live for current competitor pricing rather than relying on memory).
5. Present as a pricing table plus the reasoning for each gate, and flag any tier that looks likely to cause churn or confusion.
