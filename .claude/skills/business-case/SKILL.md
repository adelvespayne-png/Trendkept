# Business Case

## Workflow
1. Get the proposal, estimated costs (one-time + ongoing), and expected benefits (revenue, cost savings, risk reduction) - ask for real numbers rather than filling in placeholders.
2. Structure: Problem/opportunity -> Proposed solution -> Costs -> Expected benefits/ROI (with the calculation shown, e.g. payback period, NPV if relevant) -> Risks and mitigations -> Alternatives considered (including "do nothing") -> Recommendation.
3. Be honest about uncertainty in benefit estimates - a range or confidence caveat is more credible than false precision.
4. Keep it scannable for a decision-maker: executive summary up top with the ask stated plainly.
