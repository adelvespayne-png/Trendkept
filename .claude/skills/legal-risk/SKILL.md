# Legal Risk

## Workflow
1. Understand the situation/document and the jurisdiction(s) involved (risk analysis is jurisdiction-dependent).
2. Scan systematically across common risk areas: contractual obligations, IP ownership/infringement, employment/contractor classification, data privacy (GDPR/CCPA-type obligations if relevant), regulatory/licensing requirements for the industry.
3. Rate each flagged item as low/medium/high concern with the reasoning, and note which items genuinely need a qualified lawyer versus which are minor/easily fixed.
4. Never present this as a substitute for legal advice - frame outputs as "things to raise with counsel," especially for anything rated medium/high.
