# Compliance

## Workflow
1. Identify the specific regulatory domain and jurisdiction (compliance requirements vary hugely by country/state and industry - don't assume).
2. Search for current requirements rather than relying on memory, since regulations change - this is exactly the kind of query where being out of date is actively harmful.
3. Build a checklist against the specific, cited requirements found, marking each as met/not met/unclear based on what the user has shared.
4. Flag explicitly that this is not a substitute for a compliance professional or lawyer, especially for anything with real regulatory penalties attached.
