# Sop Builder

## Workflow
1. Interview the user on the actual current process (or extract it from what they describe) - don't invent steps.
2. Structure: Purpose -> Scope (who/when this applies) -> Numbered steps (one action per step, specific enough that someone unfamiliar could follow it) -> Common pitfalls/exceptions -> Owner/last-updated.
3. Use imperative voice ("Click X", "Send the email to Y") not passive description.
4. Call out any decision points explicitly ("If X, do A; if Y, do B") rather than burying branching logic in prose.
5. Keep it as short as the process allows - SOPs that are too long don't get followed.
