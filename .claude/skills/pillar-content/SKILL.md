# Pillar Content

## Workflow
1. Confirm the pillar topic and the list of cluster subtopics it should link out to (build one with content-strategy skill logic if not provided).
2. Structure: broad intro framing the whole topic -> sections matching each cluster subtopic at a summary level -> clear internal link-out point for each ("For a deep dive on X, see [cluster link]") -> conclusion.
3. Keep each section substantial enough to stand alone but not so deep it duplicates the cluster article - the pillar orients, the cluster goes deep.
4. Write real, specific content - no filler section headers with vague paragraphs.
